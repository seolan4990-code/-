신한저축은행 신디 모바일 대출신청화면· TO-BE 퍼블리싱 정의서

UI/UX 변경후 대상 화면(TO-BE) — 대상화면 ① 한도조회중(로딩화면), ② 한도조회 결과화면 —기획내용을 정리한 핸드오프 문서입니다. 개발 및 퍼블리싱 과정에서 참고하실 수 있도록 상세 명세(디자인 토큰, 컴포넌트, 인터랙션 등)를 통합 정리했습니다.(관련 궁금사항은 민설안프로에게 문의주세요.)

기준뷰포트 모바일 웹뷰 (≈380px) 개발요청화면 2종 Primary #2B5CE6 폰트 Noto Sans KR ▶ 라이브 프로토타입

Purpose

01 · 목적

Vol.1 고객이 이해하고 선택하며 스스로 관리하는 금융서비스 고객을 위해 대출은 명확하고 이해하기 쉽게 이해 간편하게 이용 할 수 있도록 화면을 개선한다.

추진과제 상세: 이해 가능성(Comprehension). 정보를 더 넣는 것이 아니라, 핵심을 쉬운 용어와 시각화로 명확하게 안내, 신청 - 심사- 실행등 대출 진행단게 직관적 UI구축, 고객이 충분히 이해한 후 적합한 대출을 선택할 수있는 모바일 대출 프로세스 제공

Direction

02 · 기획 방향 (3가지의 기준을 잡았습니다.)
  1. 금융 소비자보호 기반 화면 개선
    중요사항 이해여부 확인, 온라인 설명의무 강화, 다크패턴 가이드라인 준수 — 규제와의무를 충족하면서 고객이 실제로 이해하도록.
  2. 고객 주관식 의견 기반 개선
    대출계약체결 완료 화면의 고객만족도 조사 결과 및 주관식 설문 의견을 화면 개선에 직접 반영.
  3. 토스웹뷰토 화면 장점 추출·벤치마킹
    "이 부분이 좋아서 참고했습니다." 수준의 취지만 반영예정 — 표현·정보구조의 장점을 발췌해 적용.
방향주로 반영되는 화면
① 대출 조건·고객 부담비용·권리사항을 쉬운 용어와 시각화로 명확화 중요사항 이해여부화면
② 고객 의견한도조회~전자약정화면
③ 벤치마킹한도조회중,동의화면 등

Scope

03 · 작업 범위

비대면 대출영역 화면의 UI/UX개선예정 (레이아웃, 정보 순서, 시각적 강조, 폰트/색/여백, 핵심정보 요약, 쉬운 말 풀이, 단계 나누기 등 프론트엔드 개발).

단계화면개선 초점현재 상태

02한도조회 결과상품 선택 UX — 요약·정렬·추천 조합·규제 확인설계·프로토타입

04중요사항 이해여부이해여부 확인 UI, 온라인 설명의무 강화, 쉬운 말 풀이설계 예정
05전자약정 (전자서명)약관 위계 정리, 핵심 조건 요약, 서명 흐름 명확화설계 예정
포함: 화면의 표현·정보구조 개선(프론트엔드).  ·  레이아웃 / 정보 순서 / 시각적 강조 / 폰트·색·여백 / 핵심정보 요약 / 쉬운 말 풀이 / 단계 나누기.

화면 1 · 한도조회중

구조 · 레이아웃

조회 대기 경험(로딩) 개선 화면. 회색 화면(AS-IS)의 "시스템이 멈춘 것으로 오해받지 않도록" 흰색 배경에 깔끔한 도트 로더(또는 로딩 애니메이션) + 단계별로 바뀌는 안내 문구로 해소하고, 뒤로가기 시 이탈 방지 시트로 조회 중 실수로 이탈하는 것을 방지합니다

① 로딩 화면 디자인 중립적인 UI유지 (승인&불승인고려) 움직이는 신한 캐릭터 삭제 (AX혁신부 7/14의견 반영)
